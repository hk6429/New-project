# 借代修辭卡牌遊戲網站設計規格

日期：2026-07-12  
狀態：已完成分節確認，待使用者整體審閱

## 1. 產品摘要

本產品是一個提供臺灣七至九年級學生使用的借代修辭遊戲化學習網站。核心採「學生自主冒險＋教師課堂活動」雙軌設計，以卡牌配對為入口，逐步延伸到完整語境判讀、借代與借喻辨析、古文理解及情境運用。

第一版不要求學生註冊。自由練習進度保存在目前裝置；課堂模式使用六位活動代碼與匿名暱稱。教師能快速建立活動並查看即時及課後彙整結果。正式題庫建立 240 題，公開題至少 200 題。

## 2. 產品目標

1. 讓初次使用者在 30 秒內開始遊戲，約 2 分鐘完成新手關。
2. 讓學生理解借代依據的是相關性，不只是背誦固定詞義。
3. 讓學生能區分借代的相關性與借喻的相似性。
4. 讓教師在 2 分鐘內建立課堂活動，課後快速掌握共同迷思。
5. 建立可追溯來源、版本及審核狀態的高可信題庫。

## 3. 第一版範圍

### 3.1 納入

- 免登入學生冒險與快速練習。
- 六區冒險地圖。
- 詞義配對、例句判讀、關係分類、修辭辨析、古文理解及情境運用。
- 錯題簿、卡牌圖鑑、等級、徽章及本機進度。
- 活動代碼、匿名暱稱、教師快速開局、即時進度及課後報告。
- 題庫建立、來源管理、雙人複核、版本紀錄及爭議停用。
- 手機、平板、筆電、Chromebook 及教室投影支援。
- PWA 與有限離線功能。

### 3.2 不納入

- 完整學生註冊、跨校身分及長期個人追蹤。
- 公開聊天、好友、私訊及學生上傳頭像。
- AI 自動產題後直接發布。
- 校際大型排行榜、廣告、付費及商業訂閱。

## 5. 內容定義與判斷原則

借代是在語文中不用通常使用的本名或語詞，改以其他具有穩定相關性的名稱或語詞代替。借喻則以兩者之間的相似性建立比喻關係。

題目判斷依序檢查：

1. 文中是否確實以另一名稱代替原對象。
2. 兩者關係是現實、文化、部分、特徵、人物、器物或所屬等相關性，還是只具有相似性。
3. 完整上下文是否支持該所指。
4. 是否存在其他同樣合理的答案。
5. 出處、版本與文字是否可驗證。

多義詞不得脫離語境出題。有爭議的修辭判斷不得計分或公開發布。

## 6. 網站資訊架構

### 6.1 學生冒險端

- 首頁：繼續冒險、快速練習、加入班級賽、今日任務。
- 冒險地圖：六個學習區域及解鎖進度。
- 遊戲場：配對、判讀、分類及混合挑戰。
- 回饋頁：錯題解析、判斷線索及相似概念比較。
- 個人頁：等級、徽章、圖鑑、錯題簿、弱點雷達。
- 進度：保存在目前裝置，支援進度碼匯出與匯入。

### 6.2 教師課堂端

- 快速開局：年級、難度、題型、題數及時間。
- 等候大廳：活動代碼、匿名暱稱及加入狀態。
- 即時活動：進度、正確率、排行榜及常見錯誤。
- 課後報告：班級整體、題目分析、概念迷思及補救建議。
- 派題：依條件產生任務連結。

### 6.3 題庫審核後台

- 題目搜尋、建立、修改、停用及版本比較。
- 原文出處、教育資源、原創標記及授權狀態。
- 草稿、待查核、已複核、已發布、有爭議、已停用狀態。
- 異常答對率、爭議回報、重複題及選項失效監測。

## 7. 六區冒險地圖

1. 名稱交換所：借代詞與實際所指配對。
2. 關係研究室：理解不同替代關係。
3. 古文時空門：判讀經典文本中的借代。
4. 生活偵查局：理解校園、媒體與生活語境。
5. 修辭迷霧谷：辨析借代、借喻、象徵與轉化。
6. 文句鍛造塔：選詞填空、改寫及情境運用。

## 8. 單局遊戲流程

1. 學生選擇模式與難度。
2. 系統顯示本局單一學習目標。
3. 學生完成 4 至 8 組配對或判讀。
4. 每題提供簡短正誤回饋，不打斷遊戲節奏。
5. 每組結束後集中講解錯題與判斷線索。
6. 系統依理解與進步提供星等、經驗值及復習卡。
7. 錯題進入間隔復習佇列。

單局目標時間為 2 至 5 分鐘。

## 9. 題庫規格

### 9.1 數量與題型

| 題型 | 題數 |
|---|---:|
| 借代詞與實際所指配對 | 56 |
| 完整例句與借代解釋配對 | 56 |
| 借代關係類型判斷 | 40 |
| 借代、借喻、象徵、轉化辨析 | 48 |
| 古文語譯與文意判讀 | 24 |
| 選詞、改寫與情境運用 | 16 |
| 合計 | 240 |

### 9.2 難度

- 基礎 72 題：線索明確、常見借代。
- 進階 96 題：需結合完整語境。
- 挑戰 72 題：多義詞、修辭辨析、古文及跨句理解。

### 9.3 語料來源

- 經典文本、成語與慣用語：55%。
- 現代生活、校園與媒體語境：30%。
- 經教師審核的原創例句：15%。

不大量搬用現行教科書題目。經典原文記錄作者、篇名與版本；現代作品須確認授權，或改用自行設計的語境。

### 9.4 教學用關係標籤

1. 特徵或標誌代本體。
2. 部分代全體。
3. 人物或專名代相關事物。
4. 器物、材料或工具代活動或成品。
5. 具體事物代抽象概念。
6. 地點、所屬或相關事物代對象。

上述為教學與管理標籤，不宣稱是唯一學術分類。同一題可保存多個候選標籤，學生作答重點放在是否能說明相關性。

### 9.5 每題資料欄位

- 編號、題型、年級、難度。
- 題幹、選項、正解、解析。
- 借代詞、實際所指、替代關係、判斷線索。
- 原文上下文、作者、篇名、來源網址或書目。
- 易混淆修辭及錯誤選項設計理由。
- 授權狀態。
- 建立者、查核者、複核者、版本及日期。
- 發布狀態、答題統計及爭議回報。

## 10. 題庫品管

1. 來源查核：確認原文、作者、篇名、字詞與版本。
2. 概念查核：確認相關性而非相似性。
3. 語境查核：確認上下文足以支持答案。
4. 選項查核：排除雙重正解及無診斷力選項。
5. 雙人複核：至少兩名具國文教學背景者同意才能發布。
6. 上線監測：異常答對率或爭議達門檻時自動停止計分並重審。

系統不得讓 AI 產生的題目略過人工複核直接發布。

## 11. 視覺與互動

- 世界觀：國文冒險學院結合古今穿越。
- 氣質：文化感、探索感、青春感；不幼兒化、不補習班化。
- 元素：書頁、印章、星圖、時空門、卡牌邊框及現代資訊面板。
- 首頁只保留三個主要行動。
- 卡牌使用大字、寬間距及明確選取狀態。
- 教師儀表板優先顯示加入人數、全班正確率與共同迷思。
- 答對與答錯同時使用文字、圖示及色彩。
- 動畫與音效可關閉，所有核心操作支援鍵盤。

## 12. 資料流與異常處理

### 12.1 課堂活動資料流

1. 教師設定活動並產生六位活動代碼。
2. 學生輸入代碼與匿名暱稱。
3. 系統保存暱稱、答題、時間及臨時識別碼。
4. 教師看到彙整結果，不接觸額外個資。
5. 活動結束後依資料保留期限刪除臨時識別資料。

### 12.2 異常處理

- 斷線：本機暫存作答，恢復後補送。
- 活動代碼失效：說明原因並引導自由練習。
- 題目有爭議：標記觀察並送人工複核，不直接改答案。
- 多重正解：停止該題計分並暫時停用。
- 教師中斷：學生完成當前題組，結果轉成個人練習紀錄。

## 13. 技術架構

- Next.js App Router 與 TypeScript 建立響應式 PWA。
- IndexedDB 保存免登入進度、錯題及離線作答佇列。
- Supabase Postgres 保存題庫、活動與答題彙整。
- Supabase Realtime 支援班級即時狀態。
- 公開題庫只讀；活動資料依臨時權杖限制；後台必須驗證。
- 所有對外資料表啟用列級安全政策並採最小權限。
- Vercel 預覽部署通過驗收後才進入正式環境。
- 題庫領域模型與遊戲引擎分離，以支援後續新增其他修辭。

## 14. 開發順序

### 第一階段：內容與遊戲核心

- 題庫格式、來源及審核狀態。
- 40 至 60 題雙人查核試行題。
- 配對、判讀、解析及計分引擎。
- 學生試玩後再擴充題庫。

### 第二階段：學生冒險版

- 首頁、六區地圖、遊戲、錯題簿、圖鑑及本機進度。
- 題庫擴充至 240 題，至少 200 題通過發布。
- PWA、離線讀題及進度匯出與匯入。

### 第三階段：教師課堂版

- 快速開局、活動代碼、匿名暱稱、即時進度及課後報告。
- 斷線補送、活動逾時、惡意暱稱過濾及頻率限制。

### 第四階段：正式上線

- 題庫審核後台、版本比較、爭議停用及品質監測。
- 資安、效能、無障礙及多裝置測試。
- 一至兩個真實班級試用，修正後發布。

## 15. 測試策略

- 內容測試：來源、語境、借代判斷、答案唯一性及解析一致性。
- 單元測試：抽題、洗牌、配對、計分、難度及間隔復習。
- 整合測試：建立活動、學生加入、補送作答、彙整報告及停用題目。
- 端對端測試：完整學生單局、完整教師課堂、離線後恢復。
- 無障礙測試：鍵盤、螢幕閱讀器、色弱、200% 縮放及減少動態效果。
- 真實情境測試：手機、平板、Chromebook、筆電及教室投影。

## 16. 正式驗收標準

### 16.1 內容

- 題庫總數至少 240 題，發布題至少 200 題。
- 所有發布題均有來源或原創標記、完整解析及雙人複核。
- 不存在未處理的多重正解或修辭分類爭議。

### 16.2 學生端

- 初次使用者 30 秒內開始首局，約 2 分鐘完成新手關。
- 答錯後能查看借代詞、實際所指與判斷依據。
- 重新整理、離線及恢復連線不遺失已完成作答。

### 16.3 教師端

- 教師 2 分鐘內建立並開始活動。
- 30 名學生能穩定加入、作答並呈現在即時報告。
- 報告能指出最常錯的題目與主要概念迷思。

### 16.4 品質

- 手機、平板、筆電及投影無版面破裂。
- 鍵盤可完成所有核心操作，文字放大至 200% 仍可使用。
- 公開頁面不得取得教師管理、答案修改或學生原始資料權限。
